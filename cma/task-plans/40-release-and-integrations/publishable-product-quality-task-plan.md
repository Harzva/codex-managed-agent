# Publishable Product Quality Task Plan

Status: complete

## Priority

Priority 1

## Objective

Make the extension's release path understandable, repeatable, and easier to evaluate directly from the repo.

## Scope

- screenshot / preview inventory
- release posture and preview messaging
- release notes and root-doc discoverability
- package / publish validation path

## Tasks

### Task 1 — Screenshot Inventory

- [x] define the minimum publishable screenshot set
- [x] name the exact UI surfaces to capture
- [x] keep the first artifact lightweight and repo-visible

### Task 2 — Release Posture

- [x] clarify preview vs release expectations
- [x] ensure README and package metadata do not send mixed signals

### Task 3 — Release Workflow Notes

- [x] keep package, smoke, and changelog docs easy to discover
- [x] make the smallest repeatable VSIX release path explicit

## Exit Criteria

- repo contains a concrete screenshot / preview capture plan
- release-facing docs are discoverable from root docs
- release posture is understandable without tribal knowledge
