# Task Plans

Task plans are grouped by product area, not by date. The plans describe durable workstreams, and several of them stay active across multiple implementation passes, so topic-based folders are easier to browse than a chronological pile.

Use the numeric prefixes as a reading order:

- `00-roadmap/` - long-term roadmap, stage handoffs, and acceptance snapshots.
- `10-agent-orchestration/` - MoA, DAG workers, loops, team space, mailbox, and provider coordination.
- `20-product-surface/` - board, visibility, no-editor workflows, insights, and memory cards.
- `30-platform-runtime/` - backend/runtime migration, reliability, communication, cross-path support, and webview decomposition.
- `40-release-and-integrations/` - publishability, presentation outputs, service/bot integrations, and other outward-facing extensions.

When adding a new plan, place it in the folder for the main capability it changes. If a plan is primarily a sequencing document or a handoff, put it in `00-roadmap/` and link to the implementation plans in the topic folders.
