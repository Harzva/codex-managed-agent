# Screenshot Capture Checklist

Use this checklist when creating final README and Marketplace screenshots.

## Before Capture

- [ ] Local `codex_manager` service is running
- [ ] Extension loads without degraded-state errors
- [ ] Real thread data is visible
- [ ] UI is in a stable, non-transition state
- [ ] VS Code theme is dark and clean
- [ ] Window chrome does not expose irrelevant clutter

## Shot 1 — Main Dashboard

- [ ] Open editor-area dashboard
- [ ] Thread list is populated
- [ ] Board summary is visible
- [ ] Inspector/drawer is open and readable
- [ ] Save as `docs/screenshots/main-dashboard.png`

## Shot 2 — Board With Active Cards

- [ ] Open full board page
- [ ] Multiple cards are visible
- [ ] Card sizes are mixed and readable
- [ ] At least one meaningful active or intervention card is visible
- [ ] Save as `docs/screenshots/board-active-cards.png`

## Shot 3 — Needs Human Dock

- [ ] Expand `Needs Human`
- [ ] Show multiple urgent cards
- [ ] Ensure no clipping or oversized overlays
- [ ] Save as `docs/screenshots/needs-human-dock.png`

## Shot 4 — Sidebar or Bottom Placement

- [ ] Open the extension in sidebar or bottom panel
- [ ] Make sure the docked state is readable
- [ ] Save as `docs/screenshots/docked-layout.png`

## Optional Shots

- [ ] `docs/screenshots/insights-topic-map.png`
- [ ] `docs/screenshots/inspector-detail.png`

## Review Pass

- [ ] No blank panels
- [ ] No drag overlays or debug states
- [ ] No clipped text
- [ ] No giant empty regions that weaken the screenshot
- [ ] Files saved with final names
