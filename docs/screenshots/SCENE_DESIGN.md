# Screenshot Scene Design

Use this file to stage each final screenshot intentionally.

## Scene 1 — Main Dashboard

### Goal

Make the extension look like a serious in-editor control surface.

### Layout

- editor area open
- thread list visible on the main surface
- one selected thread
- inspector/drawer open on the side
- top chrome fully loaded and readable

### Content

- a few realistic thread titles
- one thread with useful log or conversation detail
- no empty or degraded state warnings

### Avoid

- giant blank regions
- too many tiny cards competing for attention
- temporary board-only debug elements

## Scene 2 — Board With Active Cards

### Goal

Show why the board exists.

### Layout

- board page only
- 4 to 8 cards visible
- at least two different card sizes
- enough spacing to look intentional, not sparse

### Content

- at least one active/running-looking card
- at least one attached/supporting card
- visible but not dominant `Needs Human` presence

### Avoid

- one lonely card in a huge empty board
- drag handles or drop overlays during capture
- huge oversized cards hiding the rest of the board

## Scene 3 — Needs Human Dock

### Goal

Prove intervention work is visible and manageable.

### Layout

- `Needs Human` expanded
- 2 to 4 urgent cards shown
- dock header visible
- main board still partially visible beneath or beside it

### Content

- cards with realistic action-required titles
- no card clipping
- no visual chaos from too many simultaneous alerts

### Avoid

- giant dock swallowing the entire screenshot
- empty or nearly empty urgent state
- oversized cards that read as broken layout

## Scene 4 — Docked Layout

### Goal

Show that the extension fits into normal VS Code workflows.

### Layout

- sidebar or bottom panel placement
- enough content to be readable in the docked area
- editor area still visible enough to show context

### Content

- one useful dashboard state
- some visible threads or board summary content

### Avoid

- tiny unreadable text due to extreme panel compression
- a docked surface that looks like a blank shell

## Optional Scene 5 — Insight View

### Goal

Show that insights are navigable and not decorative only.

### Layout

- topic map visible
- at least one connected guidance/report block visible

### Content

- visible nodes
- non-empty weekly shift or report content

## Optional Scene 6 — Inspector Detail

### Goal

Show depth without chaos.

### Layout

- one selected thread
- detail panel open
- readable metadata and recent context

### Content

- one good log or detail section
- no giant collapsed placeholders

## Approval Rule

If a screenshot does not make the extension look more understandable or more credible, do not use it.
